Belajar Git

git config

git config --global user.name "namamu"

git config --global user.email emailmu@gmail.com

git init

digunakan untuk membuat projek baru

  • git init . (membuat project di folder saat ini)
  • git init nama_project (membuat project dengan membuat file baru)

git status

digunakan untuk mengecek status dalam sebuah project

git add

digunakan untuk menambahkan file

  • git add nama_file (satu file)
  • git add nama_file, nama_file (beberapa file)
  • git add *.ekstensi_file (semua file ssuai ekstensi)
  • git add . (menambahkan semuanya)

git commit -m "keterangan"

git commit untuk melaukan commit, -m untuk menambahkan "keterangan pada commit yang dilakukan"

git log

untuk melihat catatan kegiatan pada projek

  • git log --oneline (melihat log singkat)
  • git log no_commit (melihat detail log pada nomor commit)
  • git log nama_file (melihat log sebuah file)
  • git log --author='nama_author' ( melihat log yang dilakukan seorang author)

git diff

melihat perubahan pada suatu commit

  • git diff no_commit (melihat perubahan file pada suatu commit)
  • git diff nama_file (melihat perubahan pada file)
  • dit diff no_commit no_commit (membandingkan perubahan dua commit)

git checkout

untuk membatalkan perubahan pada status modified(add)

git reset

untuk membatalkan perubahan pada status stagged(commit)

git branch

pembuatan cabang

  • git branch nama_branch (untuk membuat brach baru)
  • git branch (untuk cek branch yang ada)
  • git checkout nama_branch (untuk masuk ke suatu branch)
  • git merge nama_branch (untuk menggabungkan branch dengan master)
  • git branch -d nama_branch (untuk menghapus branch)

git remote

git remote add github ssh/https

https : minta password setiap kali push

ssh : tidak minta password setiap kali push, tapi harus melakukan konfigurasi ssh

  • git remote add github ssh/https
  • git remote -v (cek remote)
  • git remote rename nama_lama nama_baru (rename remote)
  • git remote remove nama

git push nama_remote cabang
